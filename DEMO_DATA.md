# AI Panel Studio — 样例演示数据

> 5 组预设话题 + 推荐嘉宾阵容，开箱即用。
> 使用方法：在 Swagger (`http://localhost:8000/docs`) 依次调用：
> 1. `POST /discussions` 创建讨论（填入 title + topic）
> 2. `POST /discussions/{id}/panelists/generate` 生成嘉宾（count 按推荐）

---

## 样例 1：AI 监管 — 政府 vs 行业

| 字段 | 值 |
|---|---|
| 标题 | AI 监管应由政府主导还是行业自律？ |
| 议题 | 围绕 AI 安全与创新平衡，探讨监管主体的最优选择 |
| 推荐嘉宾数 | 5 人 |
| 预期辩论点 | 立法必要性 vs 行业灵活性、安全底线 vs 创新空间、国内监管 vs 国际协调 |
| 预期共识 | AI 安全是不可退让的底线；需要建立统一的安全评估标准 |
| 预期分歧 | 监管力度（强监管 vs 轻监管）、立法时间表（立即 vs 渐进） |

## 样例 3：远程办公未来

| 字段 | 值 |
|---|---|
| 标题 | 远程办公是未来主流还是过渡方案？ |
| 议题 | 后疫情时代，混合办公模式能否持续？企业效率、员工福祉、城市经济三重博弈 |
| 推荐嘉宾数 | 5 人 |
| 预期辩论点 | 效率提升 vs 协作损耗、员工自由 vs 企业文化、写字楼经济 vs 分布式办公 |

| 预期分歧 | 混合比例（3:2 vs 2:3 vs 1:4）、适合远程的岗位边界 |

## 样例 4：自动驾驶责任归属

| 字段 | 值 |
|---|---|
| 标题 | 自动驾驶出事故，谁该负责？ |

| 预期分歧 | 车企承担责任的上限、驾驶员接管义务的范围、监管介入的深度 |

## 快速演示流程（10 分钟）

```bash
# 1. 确保前后端都已启动
# 2. 打开 Swagger: http://localhost:8000/docs

# 3. 创建讨论
POST /api/v1/discussions
{ "title": "AI 监管应由政府主导还是行业自律？", "topic": "围绕 AI 安全与创新平衡" }

# 4. 记下返回的 discussion_id（设为 1）

# 5. 生成嘉宾
POST /api/v1/discussions/1/panelists/generate
{ "count": 5 }

# 6. 打开前端 http://localhost:5173 → 进入讨论 1 的演播厅
#    或直接访问 http://localhost:5173/studio/1

# 7. 触发 5-8 轮发言，观察打字机效果 + 共识/分歧面板

# 8. 生成总结
POST /api/v1/discussions/1/summary
```

> 建议：第一场演示用样例 1（AI 监管），话题有争议性、容易形成共识和分歧、演示效果好。
